# `/autoplan` 技能深度注解

> 对应源文件：[`autoplan/SKILL.md.tmpl`](https://github.com/garrytan/gstack/blob/main/autoplan/SKILL.md.tmpl)
> 一条命令跑完全部评审——CEO + Design + Eng + DX。

---

## 这个技能是什么？

`/autoplan` 把手动运行 4 个评审技能的工作变成一条命令。

**触发时机**："帮我把评审全跑了，该做的决定你来做。"

**它做什么**：
- 顺序运行：CEO → Design → Eng → DX 全部评审
- 6 条决策原则自动回答中间问题
- 品味决策（合理分歧）放到最后的批准门关
- 输出：完全评审过的方案

**不做什么**：不写代码。不跳过任何评审步骤。

---

## 6 条决策原则

```
1. 选择完整性 — Ship the whole thing
2. 清空湖底 — 在爆炸半径内修一切
3. 务实 — 两个方案修同样问题？选干净的
4. DRY — 重复已有功能？拒绝
5. 显式优于聪明 — 10 行明显修复 > 200 行抽象
6. 偏向行动 — Merge > review cycles > stale deliberation
```

**冲突解决（按阶段）：**
- CEO 阶段：完整性 + 清空湖底 优先
- Eng 阶段：显式 + 务实 优先
- Design 阶段：显式 + 完整性 优先

---

## 决策分类

**机械决策**：答案明确。静默自动决定。
  例：跑 Codex？→ 永远 yes。缩减完整方案的范围？→ 永远 no。

**品味决策**：合理分歧。自动决定但在最终门关呈现。
  三种来源：
  1. 两个方案都可行但各有取舍
  2. 在爆炸半径边缘（3-5 个文件）
  3. Codex 有不同意见且有道理

**用户挑战**：两个模型都认为应该改变用户的方向。永远不自动决定。

---

## 核心流程

```
/autoplan
    │
    ▼
┌───────────────────────────┐
│ 读 4 个评审技能的完整文件 │
│ plan-ceo-review           │
│ plan-design-review        │
│ plan-eng-review           │
│ plan-devex-review         │
└───────────┬───────────────┘
            │
            ▼
┌───────────────────────────┐
│ 顺序执行每个评审          │
│ 中间 AskUserQuestion      │
│ → 6 原则自动回答         │
│ → 品味决策标记           │
│ → 用户挑战不自动决定     │
└───────────┬───────────────┘
            │
            ▼
┌───────────────────────────┐
│ Final Approval Gate       │
│ 呈现所有品味决策 + 挑战  │
│ 用户逐个确认/覆盖        │
└───────────────────────────┘
```

---

## 总结

| 设计决策 | 原因 |
|---------|------|
| 6 条自动决策原则 | 减少 15-30 个中间问题 |
| 按阶段的冲突解决 | CEO 和 Eng 的优先级不同 |
| 品味决策延迟 | 汇总到最后一次确认 |
| 用户挑战不自动 | 方向性改变必须人决定 |
| 全深度执行 | 不跳步骤，和手动一样严格 |
