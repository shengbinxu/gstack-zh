# `/autoplan` 技能深度注解

> 对应源文件：`autoplan/SKILL.md.tmpl`
> 一条命令跑完全部评审（CEO → Design → Eng → DX）。

## 核心定位

`/autoplan` = 自动评审管线。读取 CEO、设计、工程、DX 四个评审技能的完整文件，按顺序执行，用 6 条原则自动决策中间问题。品味决策（close call）集中到最后审批门。

## 6 条自动决策原则

| # | 原则 | 含义 |
|---|------|------|
| 1 | 选择完整性 | 选覆盖更多边界情况的方案 |
| 2 | Boil lakes | 爆炸半径内 + <1天工作量 → 批准扩展 |
| 3 | 务实 | 两个方案修同一个问题 → 选更干净的 |
| 4 | DRY | 重复现有功能 → 拒绝 |
| 5 | 显式优于聪明 | 10行明显代码 > 200行抽象 |
| 6 | 偏向行动 | 合并 > 反复评审 > 拖延讨论 |

## 决策分类

- **Mechanical（机械）**：明确正确答案 → 静默自动决策
- **Taste（品味）**：合理的人会有不同意见 → 自动决策但最后集中确认
- **User Challenge（用户挑战）**：两个模型都认为用户方向应该改 → 永不自动决策，必须问用户

## 顺序执行（强制）

CEO → Design（如有 UI） → Eng → DX（如有开发者界面），严格串行。每个阶段完成后才开始下一个。
