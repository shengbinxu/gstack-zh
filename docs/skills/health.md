# `/health` 技能深度注解

> 对应源文件：[`health/SKILL.md.tmpl`](https://github.com/garrytan/gstack/blob/main/health/SKILL.md.tmpl)
> 代码质量仪表盘——一个分数看全局。

---

## 这个技能是什么？

**人设**："拥有 CI 仪表盘的 Staff 工程师"。

**HARD GATE**：不修任何问题——只出仪表盘和建议，用户决定行动。

---

## 核心流程

```
/health
  │
  ▼
┌───────────────────────────────┐
│ Step 1: 检测健康栈            │
│ ├─ CLAUDE.md 有 Health Stack？│
│ │   └─ 有→用它               │
│ └─ 没有→自动检测：           │
│     ├─ tsconfig.json → tsc   │
│     ├─ eslint/biome → lint   │
│     ├─ package.json → test   │
│     ├─ pyproject.toml → ruff │
│     └─ go.mod → go test      │
├───────────────────────────────┤
│ Step 2: 运行每个工具          │
│ ├─ 类型检查                   │
│ ├─ Linter                     │
│ ├─ 测试                       │
│ ├─ 死代码检测                 │
│ └─ Shell linter               │
├───────────────────────────────┤
│ Step 3: 加权评分              │
│ └─ 复合 0-10 分              │
├───────────────────────────────┤
│ Step 4: 趋势追踪              │
│ └─ vs 上次 /health 结果      │
├───────────────────────────────┤
│ Step 5: 输出仪表盘            │
│ ├─ 每个工具的得分             │
│ ├─ 综合得分                   │
│ ├─ 趋势（↑/↓/→）            │
│ └─ 建议行动                   │
└───────────────────────────────┘
```

---

## 关键设计决策

| 决策 | 原因 |
|------|------|
| 不发明新工具 | 包装项目已有的 linter/tester/checker |
| 0-10 加权复合分 | 一个数字看全局 |
| HARD GATE 不修 | 仪表盘，不是自动修复 |
| 趋势追踪 | 质量是在改善还是在下降？ |
| 自动检测 | 不需要配置就能用 |
