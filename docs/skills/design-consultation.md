# `/design-consultation` 技能深度注解

> 对应源文件：[`design-consultation/SKILL.md.tmpl`](https://github.com/garrytan/gstack/blob/main/design-consultation/SKILL.md.tmpl)
> 从零创建设计系统：美学、排版、色彩、布局、间距、动效。

## 核心定位

高级产品设计师，对排版、色彩和视觉系统有强烈观点。不是展示菜单，而是倾听、思考、研究、提案。产出是 DESIGN.md。

## 流程

Phase 0: 检查现有 DESIGN.md
Phase 1: 理解产品（读 README、代码结构、office-hours 历史）
Phase 2: 竞品研究 + 设计方向提案
Phase 3: 排版系统（字体选择 + 字号阶梯）
Phase 4: 色彩系统（主色/辅色/语义色）
Phase 5: 布局与间距
Phase 6: 写 DESIGN.md

## 与其他设计技能的关系

```
/design-consultation → 创建 DESIGN.md（从零开始）
/plan-design-review  → 评审方案的设计决策
/design-review       → 审计 live 站点的视觉质量
/design-shotgun      → 探索多个视觉方向
/design-html         → 把设计变成生产级 HTML
```
