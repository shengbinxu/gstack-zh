# `/design-consultation` 技能深度注解

> 对应源文件：[`design-consultation/SKILL.md.tmpl`](https://github.com/garrytan/gstack/blob/main/design-consultation/SKILL.md.tmpl)
> 从零创建设计系统——设计顾问，不是表单向导。

---

## 这个技能是什么？

**触发时机**：新项目，还没有设计系统。

**它做什么**：理解产品 → 研究竞品 → 提出完整设计系统 → 生成字体+配色预览页 → 输出 DESIGN.md

**核心人设**："有强烈审美观点的资深设计师"——不给你菜单选，而是提出完整方案并解释原因。

---

---

## 关键设计决策

### DESIGN.md 作为唯一真相源

`DESIGN.md` 不只是文档——它是其他设计技能（`/design-review`、`/plan-design-review`）读取的配置。创建它意味着后续所有设计评审都有了基准。

### "设计顾问，不是表单向导"

不会问"你想要暖色还是冷色？"，而是根据产品性质提出完整方案："考虑到你做的是开发者工具，我建议用 Inter 作为正文字体，因为..."。然后欢迎推回和调整。

---

## 总结

| 设计决策 | 原因 |
|---------|------|
| 输出 DESIGN.md | 给后续技能提供设计基准 |
| 先理解再设计 | 不做没有上下文的美学选择 |
| 生成预览页 | 看到实际效果比读描述有用 |
| 不给菜单 | 设计师提方案，不是让用户自助 |
