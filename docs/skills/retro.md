# `/retro` 技能深度注解

> 对应源文件：[`retro/SKILL.md.tmpl`](https://github.com/garrytan/gstack/blob/main/retro/SKILL.md.tmpl)
> 工程周回顾——从 commit 历史生成团队洞察。

---

## 这个技能是什么？

`/retro` 分析 commit 历史、工作模式和代码质量指标，生成工程回顾报告。

**触发时机**：周末或 sprint 结束。

**它做什么**：
- 分析指定时间窗口内的所有 commit
- 团队感知：多人贡献时按人细分，每人有 praise + 成长建议
- 持久化历史 + 趋势追踪（对比上次回顾）
- 消费 /ship 写入的指标（覆盖率、发布频率等）
- 支持跨项目全局回顾

**不做什么**：不改代码，只分析和报告。

---

## 参数系统

```
/retro              默认：最近 7 天
/retro 24h          最近 24 小时
/retro 14d          最近 14 天
/retro 30d          最近 30 天
/retro compare      当前窗口 vs 上一个同长度窗口
/retro compare 14d  指定窗口对比
/retro global       跨项目全局回顾（不需要在 git 仓库内）
/retro global 14d   跨项目 + 指定窗口
```

**时间对齐**：天/周单位按本地午夜对齐（`--since="2026-03-11T00:00:00"`），
小时单位用相对时间（`--since="N hours ago"`）。不加 `T00:00:00` 后缀，
git 会用当前时钟时间，导致午夜前运行少算一天。

## 核心流程

```
/retro [window]
     │
     ▼
┌─────────────────────────────────┐
│ Step 1: 识别运行者 + 贡献者    │
│ └─ git log → 提取所有 author   │
├─────────────────────────────────┤
│ Step 2: Commit 分析             │
│ ├─ 按人分组                     │
│ ├─ 按类型分类（feat/fix/chore）│
│ └─ 按时间分布（工作模式）      │
├─────────────────────────────────┤
│ Step 3: 代码质量指标            │
│ ├─ 消费 /ship 持久化数据       │
│ │   └─ 覆盖率、plan 完成度     │
│ ├─ 测试比（测试代码/生产代码） │
│ └─ 热点文件（频繁修改的文件）  │
├─────────────────────────────────┤
│ Step 4: 按人生成               │
│ ├─ Praise（做得好的）          │
│ └─ Growth（可以更好的）        │
├─────────────────────────────────┤
│ Step 5: 趋势追踪               │
│ └─ vs 上次回顾：改善/下降/稳定│
├─────────────────────────────────┤
│ Step 6: 持久化                  │
│ └─ ~/.gstack/projects/retro-*  │
└─────────────────────────────────┘
```

## Compare 模式

```
/retro compare 14d
```

自动计算两个窗口：
- 当前窗口：最近 14 天
- 对比窗口：之前的 14 天

对比维度：commit 数量、测试覆盖率变化、热点文件变化、工作模式变化。

---

## Global 模式

```
/retro global
```

不局限于当前仓库——扫描 `~/.gstack/` 下所有项目的数据，
生成跨项目的开发活动总览。适合用一个 AI 工具管理多个项目的独立开发者。

---

## 关键设计决策

| 决策 | 原因 |
|------|------|
| 按人细分 | 团队感知，不是匿名统计 |
| Praise + Growth | 不只是数据——给建设性反馈 |
| 趋势追踪 | 不只看本周，看跨周变化 |
| 消费 /ship 数据 | 利用已有指标，不重复收集 |
| 午夜对齐 | 防止"运行时间影响结果"的 bug |
| 持久化 | 跨会话保存，下次可对比 |
| Global 模式 | 独立开发者的跨项目视图 |
